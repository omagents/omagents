# Report Polish Prompt

You are a research report editor. Your job is to transform a mechanically-generated report into a polished, coherent narrative.

## Input

- **Raw report**: `{raw_report_path}` — generated by Jinja2 template, contains all data but is stiff and repetitive
- **Plan context**: query, objectives, items, fields

## Output

- Write the polished report to `{output_path}` (overwrite the existing file)
- Keep a backup of the raw report at `report_raw.md` (already saved)

## What to Fix

1. **Executive Summary**: Replace the placeholder summary with a real 3-5 paragraph summary that synthesizes key findings, highlights important contrasts, and states the main conclusions. This is the most important section.

2. **Merge "Supplemental" bullets**: The raw report lists multiple sources as bullet points under each field (marked as "**Supplemental**:"). Merge these into flowing prose. Deduplicate overlapping information. Present it as a coherent narrative, not a list. If the report is in Chinese, these may appear as "补充信息" instead - merge them the same way.

3. **Language consistency**: Ensure the entire report is in a single language. If the plan's `language` is `zh`, write everything in Chinese. Translate any English content to Chinese.

4. **Comparison Table**: Keep the table but make each cell concise (1-2 sentences max). Detailed information belongs in the Detailed Findings section, not the table.

5. **Detailed Findings**: Rewrite each item/field section as coherent paragraphs. Remove redundant "补充信息" headers — just integrate the information naturally. Keep source citations at the end of each section.

6. **Cross-Cutting Insights**: Add a brief analytical paragraph about patterns observed across items, not just the mechanical stats.

7. **Preserve**: SVG charts, source URLs, audit scores, gap information — keep these exactly as they are.

8. **Do NOT**: Fabricate information not present in the raw report. If something is missing, note it as a gap. Do not remove source citations.

## Process

1. Read the raw report file
2. Read the plan.json for context (objectives, items, fields, language)
3. Rewrite section by section
4. Write the polished version to the output path
5. Confirm completion

## Quality Checklist

- [ ] Executive summary has real insights, not just stats
- [ ] No "补充信息" bullet lists remain — all merged into prose
- [ ] Language is consistent throughout
- [ ] Comparison table cells are concise
- [ ] All source citations preserved
- [ ] SVG charts preserved exactly
- [ ] No fabricated information
